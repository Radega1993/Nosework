## MODIFIED Requirements

### Requirement: Event detail page includes breadcrumbs
The event detail page SHALL include breadcrumb navigation using the reusable `Breadcrumbs` component that renders both visual navigation and Schema.org BreadcrumbList markup.

#### Scenario: Breadcrumbs are displayed
- **WHEN** user views the event detail page
- **THEN** breadcrumbs are displayed showing: Inicio > Eventos > [Event Title]
- **AND** breadcrumbs use the reusable `Breadcrumbs` component
- **AND** breadcrumb links are functional and use Next.js Link component
- **AND** breadcrumbs are accessible via keyboard navigation
- **AND** breadcrumbs use proper semantic HTML (`<nav>` with `<ol>`)

#### Scenario: Breadcrumbs include Schema.org BreadcrumbList
- **WHEN** breadcrumbs are rendered on the event detail page
- **THEN** Schema.org BreadcrumbList JSON-LD markup is automatically generated by the `Breadcrumbs` component
- **AND** the BreadcrumbList schema includes all breadcrumb items (Inicio, Eventos, Event Title)
- **AND** the schema uses absolute URLs for each breadcrumb item
- **AND** the schema is valid and can be tested with Google Rich Results Test

### Requirement: Event detail page includes SEO meta tags
The event detail page SHALL include optimized meta tags for SEO using the `SEOHead` component, including dynamic title, description, Open Graph tags, Twitter Cards, and canonical URL specific to the event.

#### Scenario: Dynamic SEO meta tags are present
- **WHEN** user views the event detail page
- **THEN** the page uses the `SEOHead` component for all meta tags
- **AND** the page includes a dynamic `<title>` tag with event name (e.g., "[Event Name] - Nosework Trial")
- **AND** the page includes a dynamic `<meta name="description">` tag with event summary
- **AND** the page includes Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`)
- **AND** the page includes Twitter Card tags (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`)
- **AND** the page includes a canonical URL pointing to the event detail page

#### Scenario: Canonical URL uses centralized utility
- **WHEN** canonical URL is generated
- **THEN** it uses the `getCanonicalUrl()` utility function
- **AND** the canonical URL is absolute (includes domain and uses HTTPS)
- **AND** the canonical URL follows consistent format (no trailing slash unless required)
- **AND** the canonical URL points to `/events/[id]` for the specific event

#### Scenario: Open Graph image is optimized
- **WHEN** event has an image
- **THEN** the `og:image` tag uses the event image
- **AND** if event has no image, a default image is used
- **AND** the image meets Open Graph size requirements (1200x630px recommended)
- **AND** the image URL is absolute (includes domain)
